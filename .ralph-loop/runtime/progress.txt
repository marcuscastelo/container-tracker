## Codebase Patterns
- Agent entrypoint ownership uses canonical mains in `apps/agent/src/runtime/runtime.entry.ts` (runtime) and `apps/agent/src/app/agent.main.ts` (supervisor loop); keep only externally required wrappers (`apps/agent/src/{agent.ts,supervisor.ts}`) and make them strict `wrapper -> main()` delegates.
- Linux launcher flows (`scripts/agent/run-linux.sh`, Linux E2E shim generation, and launcher tests) must target `dist/apps/agent/src/supervisor.js` directly; do not route through removed bootstrap forwarding wrappers.
- Canonical agent path names now live in `apps/agent/src/platform/agent-path-layout.ts`; when resolving platform runtime paths, call `resolveAgentPathLayoutPaths` with the OS-specific join function (`path.join` or `path.win32.join`) instead of duplicating literal file/directory names.
- Keep cross-platform path-layout drift guarded by asserting adapter `resolvePaths()` outputs against `resolveAgentPathLayoutPaths` in integration tests (`apps/agent/src/tests/platform-layout-integration.test.ts`).
- Agent source files enforce `noRestrictedImports`; use absolute `@agent/...` aliases even for sibling imports inside `apps/agent/src/**`.
- Keep local-control behavior inside `apps/agent/src/platform/control/*` strategies and keep `apps/agent/src/platform/local-control.adapter.ts` as thin strategy composition/dispatch only.
