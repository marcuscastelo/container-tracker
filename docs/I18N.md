# I18N Guide

Este repositório usa `@solid-primitives/i18n` com **locale único ativo `pt-BR`**.

## Regras canônicas

- i18n pertence à camada de UI/presentation.
- Domain, application, infraestrutura backend e capabilities não traduzem labels canônicos.
- `src/entry-client.tsx`, 404 e fallbacks leves do shell não devem puxar `shared/localization` por conveniência.
- ponto de import canônico é `~/shared/localization/i18n`.

## API pública

```ts
const { t, keys, locale } = useTranslation()
```

- `t(key, params?)`: resolve string por chave e aplica interpolação simples.
- `keys`: objeto tipado para manter ergonomia `t(keys.dashboard.header.title)`.
- `locale()`: retorna sempre `pt-BR` no estado atual do produto.

## Política de fallback

- Em `dev`: chave ausente retorna `[missing] <key>` e gera `console.warn` vez.
- Em `prod`: chave ausente retorna própria `<key>` para não quebrar renderização.

## Dicionário

- único dicionário ativo fica em `src/locales/pt-BR.json`.
- organização continua por namespaces de superfície (`dashboard`, `shipmentView`, `search`, etc.).
- Não criar barrels que importem dicionários enormes no shell global.

## Guardrails

- `pnpm run i18n:check`: valida que chaves referenciadas existem em `pt-BR.json`.
- `pnpm run i18n:guard`: bloqueia reintrodução de `i18next`, helpers legados e import de localization em `entry-client.tsx`.
